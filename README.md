# ResearchAgent

**ResearchAgent** — это автоматический агент-исследователь, который изучает заданную тему, разбивая её на подвопросы, собирает ответы с помощью LLM (через [Ollama](https://ollama.com/)) и сохраняет прогресс в Markdown-файлах. Агент умеет возобновлять работу после перезапуска, использует кэширование ответов и строит дерево задач.

## 📋 Возможности

- **Раздельное хранение**:  
  - `tasks.md` — дерево задач (вопросы с чекбоксами, вложенность).  
  - `knowledge.md` — база знаний (вопрос → ответ).  

- **Контекстные подвопросы**: агент учитывает уже полученные ответы из родительских и братских узлов, чтобы генерировать релевантные уточняющие вопросы.

- **Автоматическое исследование**: агент выбирает узел с максимальным «потенциалом незнания» (открытые вопросы или узлы с неполными поддеревьями) и отвечает на него.

- **Глубина исследования** ограничена параметром `MAX_RESEARCH_DEPTH` (по умолчанию 3).

- **Сохранение состояния**: после каждого шага обновляются `tasks.md` и `knowledge.md`. При перезапуске агент восстанавливает дерево и базу знаний.

- **Финальный отчёт** (`final_report.md`) собирает все закрытые вопросы и ответы.

## 🚀 Требования

- Python 3.8+
- [Ollama](https://ollama.com/download) с установленной моделью (например, `gemma4:e4b-it-q4_K_M` или любая другая)
- Библиотека `ollama` (Python-клиент для Ollama)

## 📦 Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/research-agent.git
cd research-agent
```

2. Установите Python-зависимости:
```bash
pip install ollama
```

3. Убедитесь, что Ollama запущен (в фоне):
```bash
ollama serve
```

4. При необходимости загрузите модель (если её нет):
```bash
ollama pull gemma4:e4b-it-q4_K_M
```

## 🧠 Использование

Запустите скрипт:
```bash
python research_agent.py
```

Введите тему исследования, например:  
`Влияние сна на когнитивные способности`

После запуска доступны команды:

| Команда | Действие |
|---------|----------|
| `s` или `Enter` | Выполнить один шаг (ответить на самый «неизвестный» открытый вопрос) |
| `a` | Автоматически выполнить 10 шагов |
| `r` | Сформировать финальный отчёт (сохраняется в `final_report.md`) |
| `q` | Сохранить прогресс и выйти |

Агент сам строит дерево подвопросов, отвечает на них и сохраняет всё в файлы.

## 📁 Структура файлов

После работы агента в директории появятся:

- `tasks.md` — дерево задач в виде маркированного списка с чекбоксами:  
  ```markdown
  # Дерево задач
  - [ ] Влияние сна на когнитивные способности
    - [x] Как сон влияет на память?
      - [ ] Какие фазы сна важны для консолидации памяти?
  ```

- `knowledge.md` — база знаний, каждый вопрос — заголовок второго уровня, за которым следует ответ:  
  ```markdown
  # База знаний

  ## Как сон влияет на память?

  Ответ:
  Сон играет критическую роль в консолидации памяти...
  ```

- `final_report.md` — плоский список всех закрытых вопросов с ответами.

## ⚙️ Настройки

В начале файла `research_agent.py` можно изменить следующие параметры:

```python
MAX_RESEARCH_DEPTH = 3          # максимальная глубина вложенности подвопросов
TASKS_FILE = "tasks.md"         # файл для дерева задач
KNOWLEDGE_FILE = "knowledge.md" # файл для базы знаний
MODEL_NAME = "gemma4:e4b-it-q4_K_M"   # имя модели Ollama (если None – определится автоматически)
```

> Примечание: если `MODEL_NAME` установлен в `None`, агент использует первую доступную модель из `ollama list`.

## 🧩 Как это работает

1. **Инициализация** – загружается дерево задач из `tasks.md` (если существует) или создаётся корневой узел с введённой темой. Загружается база знаний из `knowledge.md`.

2. **Выбор цели** – рекурсивно вычисляется «потенциал незнания» каждого узла:
   - Закрытый узел без детей → потенциал 0.
   - Открытый узел без детей → потенциал 1.
   - Узел с детьми → потенциал = среднее арифметическое потенциалов детей (с коэффициентом 0.5 для открытого узла).  
   Агент выбирает узел с максимальным потенциалом.

3. **Генерация ответа** – если ответ на вопрос уже есть в `knowledge.md`, он используется. Иначе вызывается Ollama с системным промптом «Ты – полезный помощник для исследований». Ответ сохраняется в базе знаний и узле.

4. **Генерация подвопросов** – если глубина узла меньше `MAX_RESEARCH_DEPTH`, агент строит контекст из ответов родителей и братьев, после чего запрашивает у LLM 3–5 уточняющих подвопросов. Новые вопросы добавляются как дети текущего узла (если они ещё не встречались в дереве или базе знаний).

5. **Сохранение состояния** – после каждого шага перезаписываются `tasks.md` и `knowledge.md`.

6. **Завершение** – когда все узлы закрыты (потенциал корня = 0), агент строит финальный отчёт.

## 🛠 Возможные проблемы

- **Модель не найдена** – проверьте, что Ollama запущен и модель загружена (`ollama pull <model>`).
- **Кодировка** – все файлы сохраняются в UTF-8. Убедитесь, что ваша консоль поддерживает русские символы.
- **Глубокие деревья** – при большой глубине возможно накопление длинного контекста. Лимит длины ответа не установлен, но вы можете сократить `MAX_RESEARCH_DEPTH`.
